# Research summary

This is a condensed pointer back to the source research document
(`DFIR_AgentBuilder_Research.docx`) and how each section of it shaped a
specific decision in the codebase.

| Research § | Topic | Decision in this codebase |
|------------|-------|----------------------------|
| §1.1 Config-repack | Velociraptor patches a YAML blob into a reserved 80KB section of its prebuilt EXE | **Diverged.** Each build is a fresh `cargo build` with config baked in via `include_bytes!`. No 80KB ceiling. No shared signature. Trade-off: 30-90s build vs ~5s repack. |
| §1.2 CreateCollector | Spec parameters (artifacts, target, encryption, opt_*) | Surfaced as `BuildSpec` in [`crates/builder-app/src/spec.rs`](../crates/builder-app/src/spec.rs) and the 6 wizard steps. |
| §1.3 VQL execution | VQL = streaming row producer | Not reused — too heavy. Each artifact module in [`collector/src/artifacts/`](../collector/src/artifacts/) is a Rust function returning `Result<ArtifactStats>`. |
| §1.4 80KB embed limit | Switching to "Generic Collector" mode at >60KB | Not applicable here — we have no embed size limit. The wizard skips the warning. |
| §1.5 Tool embedding | `winpmem.exe` packaged inside the binary | MVP ships winpmem as a sibling file (the collector invokes `./winpmem.exe`). Embedding via `include_bytes!` + extract-on-run is a 1-day follow-up. |
| §2.1-2.10 Artifact catalog | Full forensic catalog | Defined as YAML under [`artifacts/`](../artifacts/) (`<platform>/<category>/*.yaml`), loaded by [`crates/builder-app/src/backend/artifact_catalog.rs`](../crates/builder-app/src/backend/artifact_catalog.rs). ~66 artifact definitions across Windows + Linux. |
| §3.1 Upload targets | S3 / GCS / Azure / SMB / SFTP / WebDAV / Local | MVP supports S3 + Local. SFTP slot is in the UI but not wired through the collector yet. |
| §3.2 S3 SigV4 with embedded keys | Direct PutObject | Implemented in [`collector/src/upload/s3.rs`](../collector/src/upload/s3.rs) — hand-rolled SigV4 to keep binary lean. Multipart for >100MB. |
| §3.3-3.4 IAM policies | Write-only collector IAM, deny non-HTTPS, deny delete | Generated by [`generate_iam_policy()`](../crates/builder-app/src/backend/aws.rs). Documented in [docs/aws-setup.md](aws-setup.md). |
| §3.5 Transfer Family SFTP | Embeds SSH key not AWS key | Roadmap. Highest-security option. |
| §3.6 Object key collision | `{user}/{site}/{host}/{ts}_{uuid}` | Implemented as `{accessKeyId}/{site}/{host}/{filename}` — `accessKeyId` substitutes for `${aws:username}`. |
| §4.1-4.2 X509 hybrid encryption | RSA-OAEP wrap + AES container | Implemented exactly in [`collector/src/crypto/x509.rs`](../collector/src/crypto/x509.rs). Container layout documented in [docs/decrypt.md](decrypt.md). |
| §5.1 Tech stack | React+Electron + Python FastAPI + Velociraptor binary | **Diverged.** Single Rust Cargo workspace — `egui` desktop wizard + Rust collector, no HTTP server / localhost. Same UX, no Python/Node/Velociraptor dependency. |
| §5.2 6-step wizard | Target → Artifacts → Upload → Encryption → Performance → Review | Implemented exactly. See [`crates/builder-app/src/ui/`](../crates/builder-app/src/ui/) (`step1_target.rs` … `step6_review.rs`). |
| §5.3 Backend build pipeline | Spawn `velociraptor.exe collector` | **Diverged.** We spawn `cargo build --release`. The "config" file is `embedded_config.json`, not `collector_spec.yaml`. |
| §5.4 Hardening | Admin check, opt_delete_after, code signing | Admin check ✓ (manifest + runtime). delete_after_upload ✓. Code signing — manual via signtool today, automated AzureSignTool integration on roadmap. |
| §5.5 GPO deployment | Startup script as SYSTEM | Documented in main README. |
| §6 Phased roadmap | 8-week implementation plan | Phase 1+2 covered by this MVP. Phase 3 (AWS infra wizard) is partial — IAM + bucket policy generators exist; KMS+Object Lock automation is roadmap. Phase 4 (load test, EDR bypass) is post-MVP. |
| §7 Risks | Embedded key exfil, EDR block, multipart partial fail | Mitigated: write-only IAM (Object Lock makes leaked key non-destructive). Multipart retry-3-times-with-backoff in `s3.rs`. EDR coexistence — manual code-signing today. |
